# Converter Design Pipeline — Analysis & Improvement Roadmap

## 1. Executive summary

- **The pipeline is power-stage-only by construction.** Stage-4 review explicitly declares control loop, gate drive, protection, EMI/input filter, and PCB layout OUT OF SCOPE (`full_design.py:998-1002`) — and those five domains are not merely unreviewed, they are entirely *unmodeled*. The KB carries the full design discipline for every one of them, and the named reviewers (Ray/Nicola) block on exactly these gaps across hundreds of trainer-lesson runs.
- **The single most expensive stage's verdict is thrown away.** Stage-4's Ray+Nicola panel computes a REJECTED/APPROVED decision (`reviewer_panel.PanelResult.decision`) that `full_design._stage4_adversarial_review` never reads — the returned `DesignOutcome` carries forward `outcome.*` and appends only a review *count* diagnostic (`full_design.py:1013-1024`, confirmed). A REJECTED design is returned indistinguishably from an APPROVED one. The capability to gate on it already exists and is correctly used in the *other* pipeline (`reviewed_stage.py:132`).
- **A large second wave of stages was built, unit-tested, and never wired in.** `cross_check`, `refinement`, `resonant_freq`, `spice_sim` have zero non-test production consumers. The consequences are concrete: `estimators_agree` is permanently UNAVAILABLE; the "closed-loop" designer is one forward pass, not a fixed point; resonant fsw is never selected by the FHA gain law; and `stage3_realize` still simulates inline. This is the **highest leverage-per-effort cluster** — mostly wiring, not rewrites.
- **The designer lacks CRE's verification rigor.** It shares CRE's runner/parasitics/realism gate but simulates only op0 at the regulation target, measures no waveforms, derives stress analytically, never flags its sim `is_closed_loop` (so regulation is silently NOT_APPLICABLE), and never triangulates analyst vs sim. CRE's two-phase methodology catches converged-but-wrong decks the designer cannot see.
- **Soft-switching and isolated/resonant design math is asserted, not computed.** ZVS is a hardcoded `zvs=True` that zeroes switching loss (`analyst.py:800-801`); resonant peak current is a fixed π/2 factor independent of m/Q; PSFB pins a hardcoded `D_eff=0.7`; transformer reset is only a D<0.5 proxy. These produce optimistic-but-passing designs for the hardest, highest-value topologies — a silent-optimism failure that contradicts fail-loud.
- **Fail-loud has a hole in BOM assembly.** `assemble_bom_from_tas` returns the TAS untouched (no raise) for whole input classes — missing stress deriver, `fsw<=0`, `pout<=0` (`assemble.py:459-473`) — so PFC/Vienna and malformed specs slip through as INCOMPLETE rather than failing loudly.
- **No pipeline-level proof.** There is no end-to-end test running the designer through real MKF+LLM to a reviewed PASS/FAIL; `tests/property/` and `tests/realism/` are empty; the 47-entry regression corpus yields INCOMPLETE on every populated entry (no real component data). The wiring fixes above would land unverified at the pipeline level.
- **What's genuinely strong:** magnetics math stays in MKF end-to-end (no `B_sat·N·A_e/L` re-implementation in the selection/Isat paths — the contested H1 formula gap was verified RESOLVED at `extract.py:278`), the two-layer engine/bounded-LLM split is clean, fail-loud is pervasive in the realize spine and sweep, and the realize chain is genuinely single-sourced across both designer pipelines.

## 2. Current architecture (as-is)

There are **two distinct converter-design orchestrators** sharing a common Stage-3 realize chain but diverging on how `fsw` and the magnetic are chosen.

### 2a. `full_design.full_design` — multi-topology screen (designs at a fixed/seeded fsw, does NOT sweep)

- **Stage 1 — dual-path topology screen.** Static `topology_id.feasible()` + an LLM topology-selector, reconciled by union with a Jaccard>0.5 warning.
- **Stage 2 — parallel fast-Pareto magnetic pick** per topology via `ProcessPoolExecutor`. Transformer/resonant families take the slow converter-design adviser (with a tier-2 full-catalogue fallback); single-inductor families take `select_fast_by_isat_margin`. Magnetics are designed entirely in MKF via `bridge.design_converter_components`; the *achieved* inductance is harvested back (`_harvest_authoritative_inductance`, `bridge.py:1090`) and re-stamped as `desiredInductance` so stress/sim/analyst stay coherent.
- **Stage 3 — REALIZE** (`stage3_realize`): `design_converter_components` → `decompose_from_spec` (ngspice deck + stencil-decomposed TAS) → `attach_components_to_tas` → `assemble_bom_from_tas` (real FET/diode/cap from the internal DB) → `enrich_tas_for_realism` → `inject_parasitics` → `simulate_closed_loop`/`simulate_steady_state` → `run_analyst` → `realism_gate.evaluate`. **Simulates only `ops[0]` at the single vout target** (`full_design.py:579-596`).
- **Stage 3b — analytical gatekeeper.** Tight-margin checks driven by a separate hardcoded `_TIGHT_MARGIN_CHECKS` dict (`full_design.py:627`). Objects only on `verdict=='fail'`; UNAVAILABLE is a warning.
- **Stage 4 — Ray + Nicola adversarial LLM panel** on the best survivor only. Raises `FullDesignError` if a reviewer LLM cannot produce a verdict — but a cleanly-returned REJECTED verdict is **purely informational** (see G01).
- **Stage 5 — teacher** stores lessons (reads `gatekeeper.approved`, the analytical Stage-3b gate, `teacher.py:247` — not the LLM panel).

### 2b. `converter_designer.design_converter` — closed-loop fsw co-design

`B2 topology_constraints.propose` (LLM, bounded+DB-checked) → `B0 converter_spec_build.build` → `B4 frequency_sweep.sweep` (MKF re-derives L ∝ 1/fsw per fsw; golden-section argmin of magnetic+switching total loss) → `B5 magnetic_picker` → `B7 op_reconcile` across all OPs → then it **reuses `full_design.stage3_realize`** with the magnetic *pinned*, re-realizes once with SPICE knobs from the real BOM (`converter_designer.py:282-286`), then Stage-3b + Stage-4. **It is a fixed two-pass realize, not a converged fixed point** — `refinement.refine()` is built but never called (see G03). Reachable only via the async `POST /jobs/design/closed-loop`.

### 2c. Shared-stages layer (`heaviside/stages/`, 19 modules)

Per `docs/stages_refactor_roadmap.md`: each stage is a deterministic engine + optional bounded-LLM layer (select-among/calibrate, never fabricate). All 8 Tier-1/Tier-2 stages exist, plus the converter-designer B-stages and the review-boundary pair (`reviewer_panel`, `reviewed_stage`). **Consumption is thin and uneven:** `converter_spec_build`, `frequency_sweep`, `op_reconcile`, `topology_constraints`, `topology_id`, `realism_gate`, `reviewer_panel`, `reviewed_stage`, `reporter` are wired in; `spice_sim`, `cross_check`, `resonant_freq`, `refinement`, `component_match`, `design_artifact`, and `stress_extract` (as a direct designer caller) have **zero non-test production consumers**.

### 2d. CRE/RE pipeline (the verification benchmark)

`re_pipeline.run_re_pipeline` shares the *same* runner, parasitics, and realism gate as the designer, but its testbench does **two-phase simulation**: Phase 1 an MKF "ideal" scaffold with real Rds_on/DCR injected and a >30pp ideal-gap abort (`re_testbench.py:1159`); Phase 2 patches the real-BOM L/C and re-simulates at *every* claimed load point (`re_testbench.py:1215-1246`); Phase 3 measures real waveforms via `.meas` and cross-checks ripple against analytical expectation in a 0.5×–2× band (`re_testbench.py:304-466`), then derives per-component stress gated on that check passing.

### 2e. What it emits today

A power-stage BOM (FET/diode/cap/controller from the internal DB) plus synthesized auxiliaries (Cin, feedback divider, bootstrap cap, Vcc bypass, soft-start cap, current-sense resistor) — but only buck-family/evidence-gated. Control loop, gate-drive IC, protection, EMI filter, and PCB layout are explicitly out of scope.

## 3. What works well

- **Magnetics math lives in MKF, end-to-end.** L is derived by MKF and re-harvested (`L_authoritative`), Isat comes only from `PyOpenMagnetics.calculate_saturation_current[_at_operating_point]` with no analytical fallback (`bridge.py:1505-1539`), core+winding losses read MKF MAS outputs (`analyst._loss_at_output`), and `frequency_sweep` re-derives the magnetic via `bridge.design_magnetics_at_fsw` inside the loop. The contested H1 analytical-Isat formula is **verified removed** — `extract.py:278` refuses the `B_sat·N·A_e/L` fallback and uses the PyOM operating-point function. This honors the all-magnetics-in-MKF rule cleanly.
- **Fail-loud discipline is real and pervasive in the realize spine.** `stage3_realize` raises `RealizeError` on every step; `frequency_sweep` raises `FrequencySweepError` rather than clamping to a band edge; `op_reconcile` raises `InfeasibleAtOP`; `topology_constraints.validate` raises on out-of-band duty/Vds or no-real-switch; Stage-4 raises `FullDesignError` on reviewer-LLM failure. `stress.py` raises `StressDerivationError` on every missing required field and returns `None` (not a guess) for unported topologies.
- **The realize chain is genuinely single-sourced.** The closed-loop designer reuses `stage3_realize` verbatim with `pinned_main`, so BOM/sim/realism/review logic is not duplicated across the two pipelines.
- **Clean two-layer separation per the roadmap.** Deterministic engines own physics; bounded LLM layers (topology-selector, constraint-proposer, magnetic-pareto-picker, `realism_gate.explain`, Ray/Nicola) only prune/select/seed and never emit a load-bearing number. `cross_check` even encodes a sharp insight — it *refuses* to triangulate analyst-vs-MKF magnetic loss because they are the same underlying number, preventing manufactured confidence.
- **The realism gate's verdict taxonomy is honest.** Every applicable check is PASS/FAIL/NOT_APPLICABLE/UNAVAILABLE; a PASS structurally requires at least one genuine physics check to pass (audit/meta checks can only FAIL); a missing-input design reads INCOMPLETE, not a false PASS (`realism.py:954-967`). Provenance is gated — every stamped real part must carry `{producer, method, source_ref, inputs_hash}`.
- **Thermal is the one cross-cutting domain that IS implemented.** `analyst.py` computes a real itemized loss budget (conduction, switching via Qg, diode Vf+Qrr, MKF core+copper, cap ESR), stamps `Tj = Tamb + loss·Rth_ja`, and sizes the required heatsink `Rth_sa` when the no-heatsink Tj exceeds a 15%-derated target (`analyst.py:338-414`).
- **CRE's two-phase methodology is a genuine falsification step** the designer lacks: ideal scaffold first, >30pp gap abort, then real-BOM patch with multi-load-point sim and a waveform/analytical cross-check that gates sim-derived stress.
- **Named-reviewer convention honored** (Ray engineering + Nicola quality) at both Stage-3b analytical gatekeeping and the Stage-4 LLM panel, with per-reviewer UI progress in the closed-loop path.
