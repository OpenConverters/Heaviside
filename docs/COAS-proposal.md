# COAS — Converter Agnostic Structure (proposal)

Wraps TAS at the converter-product level. Mirrors MAS/CAS/SAS/RAS pattern: `inputs` + payload + `outputs`. **Does not modify TAS schema.**

## Principle

Anything derivable from `(inputs, converter)` belongs in `outputs[]`, not as nameplate. `outputs[i]` must be reproducibly generated by running `inputs.operatingPoints[i]` against `converter.tas`.

## Shape

```
COAS/schemas/
├── COAS.json                            # { inputs, converter, outputs } — all required
├── converter.json                       # manufacturerInfo, distributorsInfo, substitutesInfo,
│                                        # certifications, revision, references, tas
├── inputs.json                          # { designRequirements, operatingPoints[] }
├── inputs/designRequirements.json       # full design contract (see below)
├── outputs.json                         # outputBase + efficiency + regulation + ripple +
│                                        # transient + losses + thermal + reliability
└── utils.json                           # thin shim → PEAS/utils.json
```

## `inputs.designRequirements` — must be sufficient to design from scratch

### electrical
- `inputSource`: `{ kind: AC|DC, voltage{min,nom,max}, frequency{min,max}?, sourceImpedance?, holdupTime?, inrushLimit? }`
- `outputs[]`: `{ voltage{min,nom,max}, current{min,nom,max,peak}, polarity, isolatedFromInput, groupReturn?, sequencing{rank,rampRate}?, regulation{line,load,cross}, ripple{p2pMax,rmsMax,bandwidth}, transient{stepLoad,dIdtMax,deviationMax,recoveryTime} }`
- `power`: `{ continuous, peak, peakDuration }`

### isolation
- `required`, `workingVoltage`, `withstandVoltage`, `class∈{functional,basic,reinforced}`, `pollutionDegree`, `overvoltageCategory`, `creepageClearanceClass`

### efficiency
- `targets`: full-load, 50%, 10%, no-load input power, standby
- `mandate?`: `DoE-VI | CoC-Tier2 | 80PLUS-{Bronze,Silver,Gold,Platinum,Titanium} | custom`

### formFactor
- `maxDimensions{L,W,H}`, `maxMass`
- `mounting∈{PCB-SMT,PCB-THT,chassis,DIN,brick-full,brick-half,brick-qtr,brick-eighth,brick-sixteenth,openFrame,enclosed}`
- `cooling∈{natural,forced(CFM),conduction,baseplate(Tmax),liquid}`

### environmental
- `temperature{ambientMin,ambientMax,storageMin,storageMax}`
- `humidity{min,max,condensing}`
- `altitudeMax`, `vibrationClass`, `shock`, `ipRating`, `conformalCoating`, `saltFog`

### compliance[]
- Each: `standard, edition, class?, level?`
- Families: safety (IEC 62368/61010/60601/61851/…), EMC (CISPR 32 A|B, FCC-15), surge (61000-4-5), ESD (61000-4-2), burst (61000-4-4), harmonics (61000-3-2 class A/C/D), radiated/conducted limits

### reliability
- `target{MTBF|FIT}`, `lifeHours`, `standard∈{MIL-HDBK-217F,Telcordia-SR-332,IEC-62380,Siemens-SN29500}`, `useEnvironment∈{GB,GM,AIC,NS,…}`

### controlInterface
- `enable`, `syncIn`, `syncOut`, `pmbus{address,revision}?`, `i2c?`, `psi?`, `svid?`, `margining`, `telemetry∈[V,I,T]`, `trim`, `currentShare`, `remoteSense`

### protections[]
- Each: `kind∈{OVP,OCP,OTP,SCP,UVLO,OPP}, threshold, tolerance, mode∈{hiccup,latch,autorestart,foldback,constantCurrent}`

### bom
- `targetUnitCost`, `maxComponentCount`, `bannedSubstances`, `preferredSuppliers`, `secondSourceRequired`, `aecQGrade?`

### topology (optional constraint)
- `forced?`, `allowedFamilies[]?`

### magnetics (optional constraint)
- `maxCoreVolume`, `allowedMaterialFamilies[]`, `maxTempRise`, `shieldedRequired`

## `inputs.operatingPoints[]`

Each: `{ inputVoltage, outputCurrents[], ambientTemperature, dutyProfile? }`. 1:1 with `outputs[]` in the COAS document.

## `outputs[i]` (derivable — must match operatingPoints[i])

`outputBase`, `efficiency`, `lineRegulation`, `loadRegulation`, `outputVoltageRipple{p2p,rms,bw}`, `transientResponse{stepLoad,deviation,recovery}`, `inputCurrentTHD`, `powerFactor`, `lossBreakdown{magnetic,semi,cap,control,snubber,total}`, `componentTemperatures[]`, `reliability{predictedFIT,predictedMTBF,standardUsed}`.

## `converter` (declared — not derivable)

Two anchor fields at top level: **`tas`** (unmodified TAS) and **`manufacturerInfo`** (finished-good identity). Everything else is grouped into four structured sub-objects:

- **`commerce`** — `distributors[]`, `substitutes[]`, `listPrice`, `leadTimeWeeks`, `minimumOrderQuantity`
- **`qualification`** — `certifications[]` (granted by notified body: standard, certificateNumber, issuer, dates, scope), `declarationsOfConformity[]`, `testReports[]`
- **`lifecycle`** — `status` enum (preview|prototype|production|nrnd|lastTimeBuy|obsolete), `releaseDate`, `lastTimeBuyDate`, `endOfProductionDate`, `revision{id,date,changeNotes,previousRevisionId}`, `previousRevisions[]`
- **`documentation`** — `references[]` (title, url, kind enum, publisher, publishedDate, documentId)

No flat metadata keys; no bare `"type": "object"` anywhere.

## Open questions (resolved)

1. ~~TAS scope inside COAS~~ → **resolved: strip (no duplication).** TAS root has no `manufacturerInfo`/`certifications`/`references`/`revision`/`compliance` fields at the schema level, so embedding TAS verbatim under `converter.tas` is automatically "stripped". No TAS schema change needed.
2. ~~Lift shared utils to PEAS~~ → **resolved: yes.** PEAS/utils.json now defines `currencyAmount`, `isolationClass`, `overvoltageCategory`, `pollutionDegree`, `mountingStyle`, `coolingMethod`, `environmentalQualification`, `complianceTarget`, `protectionMode`, `reliabilityStandard`. COAS references them; MAS/CAS/SAS/RAS may be migrated to reference them in a follow-up.

## Dropped as derivable

schematicSvg, pinFunctions, componentCount, bomCost rollup, peak/typical efficiency, noLoadInputPower, holdUpTime (unless explicitly a requirement), riseTime/turnOnDelay, mtbf rollup, simulationModels, controlSoftware blob.
