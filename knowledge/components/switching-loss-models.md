## Overview: Why Switching Losses Are Critical

Switching losses are the #1 source of efficiency prediction errors in power electronics. They are difficult to calculate accurately because:

1. Device capacitances are highly nonlinear (Coss varies 10-100x over voltage range)
2. Datasheet parameters are specified at specific test conditions that may not match your circuit
3. Parasitic inductances (especially common source inductance) profoundly affect real switching waveforms
4. Reverse recovery behavior is temperature-dependent and often underspecified
5. Simple "textbook" formulas can be off by 2-3x compared to reality

This guide presents multiple models in order of increasing accuracy, with guidance on when each applies.

---

## 1. Total Switching Loss Breakdown

For any hard-switched power MOSFET, total switching loss has these components:

```
P_sw_total = P_cross_on + P_cross_off + P_Coss + P_Qrr + P_gate + P_dead
```

Where:
- P_cross_on:  Turn-on V-I overlap (crossover) loss
- P_cross_off: Turn-off V-I overlap (crossover) loss
- P_Coss:      Output capacitance charging/discharging loss
- P_Qrr:       Reverse recovery loss (from freewheeling diode or complementary FET body diode)
- P_gate:      Gate drive loss
- P_dead:      Dead time body diode conduction loss (half-bridge)

Each is discussed in detail below.

---

## 2. Model A: Piecewise-Linear Overlap (Simplest)

**Source:** Erickson & Maksimovic Ch. 4; Billings & Morey; most introductory textbooks

### Equation

```
P_cross = 0.5 * V_DS * I_D * (t_on + t_off) * f_sw
```

### Terms

| Symbol | Definition | Units |
|--------|-----------|-------|
| V_DS   | Drain-source voltage during off-state (approximately V_bus) | V |
| I_D    | Drain current at switching instant | A |
| t_on   | Turn-on transition time (current rise + voltage fall) | s |
| t_off  | Turn-off transition time (voltage rise + current fall) | s |
| f_sw   | Switching frequency | Hz |

### Assumptions

- Linear ramps of voltage and current during transitions
- Instantaneous diode commutation (no reverse recovery)
- Voltage and current transitions are simultaneous and overlapping
- t_on and t_off taken from datasheet rise/fall times (tr, tf)

### Accuracy

- Typically **overestimates** loss by 30-100% because actual waveforms are not triangular overlaps
- Does not account for the fact that current rise and voltage fall are sequential, not simultaneous
- Does not include Coss or Qrr losses
- Useful only for rough first-pass estimates

### When to Use

- Initial topology comparison where exact losses do not matter
- Educational purposes
- When datasheet provides only tr and tf (no gate charge data)

## 3. Model B: Gate-Charge-Based (Recommended for Design)

**Source:** TI SLYT664; Infineon "MOSFET power losses calculation using the Data Sheet parameters"; EPC AN030

This model uses gate charge parameters to derive transition times, which is more accurate than using datasheet tr/tf because gate charge is relatively independent of test circuit conditions.

### Turn-On Switching Loss

The turn-on transition has two sequential phases:

**Phase 1: Current rise** (VDS still high, ID rising from 0 to I_load)

```
t_1 = Q_GS2 / I_G_on
```

During this phase, VDS is approximately constant at V_bus. The energy dissipated:

```
E_on_1 = 0.5 * V_bus * I_load * t_1
```

**Phase 2: Voltage fall** (ID = I_load, VDS falling from V_bus to ~0)

```
t_2 = Q_GD / I_G_on
```

During this phase (the Miller plateau), ID is approximately constant at I_load. The energy dissipated:

```
E_on_2 = 0.5 * V_bus * I_load * t_2
```

**Total turn-on crossover energy:**

```
E_cross_on = 0.5 * V_bus * I_load * (Q_GS2 + Q_GD) / I_G_on
```

### Turn-Off Switching Loss

The turn-off transition reverses the sequence:

**Phase 1: Voltage rise** (ID = I_load, VDS rising from ~0 to V_bus)

```
t_3 = Q_GD / I_G_off
```

**Phase 2: Current fall** (VDS = V_bus, ID falling from I_load to 0)

```
t_4 = Q_GS2 / I_G_off
```

**Total turn-off crossover energy:**

```
E_cross_off = 0.5 * V_bus * I_load * (Q_GS2 + Q_GD) / I_G_off
```

### Combined Switching Power Loss

```
P_cross = [E_cross_on + E_cross_off] * f_sw
       = 0.5 * V_bus * I_load * (Q_GS2 + Q_GD) * [1/I_G_on + 1/I_G_off] * f_sw
```

### Gate Drive Current

```
I_G_on  = (V_drv - V_plateau) / R_G_on_total
I_G_off = (V_plateau - V_drv_off) / R_G_off_total
```

Where:
- V_drv: Gate driver output high voltage (e.g., 10-12 V for Si, 5-6 V for GaN)
- V_drv_off: Gate driver output low voltage (0 V for Si, -3 V for GaN)
- V_plateau: Miller plateau voltage (approximately V_th + I_load / g_fs)
- R_G_on_total: Total gate resistance during turn-on = R_G_driver_on + R_G_internal
- R_G_off_total: Total gate resistance during turn-off = R_G_driver_off + R_G_internal

### Key Parameters

| Symbol | Definition | Where to Find |
|--------|-----------|---------------|
| Q_GS2  | Gate charge from V_th to start of Miller plateau | Datasheet Qg curve, or Q_GS - Q_GS1 |
| Q_GD   | Gate-drain (Miller) charge | Datasheet, specified directly |
| V_plateau | Miller plateau voltage | Datasheet Qg curve, or calculate from V_th + I_D/g_fs |
| R_G_int | Internal gate resistance | Datasheet |
| g_fs   | Transconductance | Datasheet transfer characteristic |

### Important Notes on Q_GS2

- Q_GS2 = Q_GS - Q_GS1, where Q_GS1 is the charge to reach V_th
- Q_GS2 changes approximately proportionally with drain current (higher I_D requires higher V_GS above threshold, so more Q_GS2)
- Q_GS2 at a different current: Q_GS2(I_new) ~ Q_GS2(I_rated) * (I_new / I_rated)
- Q_GD is relatively constant with drain current but changes with V_DS (higher V_DS = more Q_GD due to nonlinear C_GD)

### Accuracy

- Typically within 20-30% for Si MOSFETs at the specified datasheet conditions
- Less accurate at conditions far from datasheet test conditions
- Does not include common source inductance effects (which slow down switching)
- Does not include Coss or Qrr losses (add separately)

### When to Use

- Design-level efficiency calculations for hard-switched converters
- Comparing MOSFETs from different vendors
- Optimizing gate drive resistance

## 4. Output Capacitance (Coss) Losses

**Source:** EPC AN030; Infineon Coss FAQ; Erickson & Maksimovic Ch. 4

### Why E = 0.5*C*V^2 Is WRONG

MOSFET output capacitance (Coss = Cds + Cgd) is highly nonlinear -- it can vary by 10-100x between V_DS = 0 and V_DS = V_rated. Using the simple capacitor energy formula with any single value of Coss gives incorrect results.

### Correct Approach: Energy Integral

The energy stored in Coss when charged from 0 to V_DS:

```
E_oss = integral from 0 to V_DS of [Coss(v) * v] dv
```

This is NOT the same as 0.5 * Coss * V_DS^2 for any single value of Coss.

**Example magnitude:** For a typical 600V superjunction MOSFET:
- Coss at 25V: ~2000 pF
- Coss at 400V: ~30 pF
- E_oss at 400V from integral: ~15 uJ
- E_oss using 0.5*Coss(400V)*400^2: ~2.4 uJ (6x too low!)
- E_oss using 0.5*Coss(25V)*400^2: ~160 uJ (10x too high!)

### Infineon Effective Capacitance Definitions

Because Coss is nonlinear, Infineon defines two equivalent linear capacitances:

**Co(er) -- Energy-related effective output capacitance:**

```
Co(er) = 2 * E_oss / V_DS^2
```

A fixed capacitor with value Co(er) stores the same energy as the nonlinear Coss when charged to V_DS. Use Co(er) for loss calculations.

**Co(tr) -- Time-related effective output capacitance:**

```
Co(tr): fixed capacitor that gives the same charging TIME as the nonlinear Coss
        when charged from 0 to V_DS with a constant current source
```

Use Co(tr) for dead time calculations in ZVS converters. Co(tr) > Co(er) always.

### Coss Loss in a Hard-Switched Half Bridge

In a half bridge, the Coss-related losses per cycle include BOTH the energy stored in the turning-on FET's Coss (which is dissipated in its channel) AND the energy drawn from the bus to charge the turning-off FET's Coss.

**Per EPC AN030, the total Coss energy dissipated per switching cycle:**

```
E_Coss_total = V_bus * Q_oss(V_bus) - E_oss(V_bus) + E_oss(V_bus)
             = V_bus * Q_oss(V_bus)    [for symmetric half bridge, NOT 2*E_oss!]
```

Wait -- this requires careful accounting. The complete breakdown per EPC AN030:

**During turn-on of Q1 (switch node rising):**
- Energy stored in Q1's Coss (which was charged to V_bus) is dissipated in Q1's channel: E_oss(Q1)
- Q2's Coss is being charged from 0 to V_bus. The bus provides V_bus * Q_oss(Q2), of which E_oss(Q2) is stored and [V_bus * Q_oss(Q2) - E_oss(Q2)] is dissipated in Q1's channel.

**Total Coss-related dissipation per switching cycle (symmetric half bridge, Q1 = Q2):**

```
E_Coss_total = E_oss + (V_bus * Q_oss - E_oss)
             = V_bus * Q_oss
```

**Power loss:**

```
P_Coss = V_bus * Q_oss(V_bus) * f_sw
```

This is ALWAYS greater than 2 * E_oss * f_sw. The difference is the resistive loss in the charging path.

### Coss Loss in Single-Switch Topologies (Buck, Boost, Flyback)

For single-switch topologies with a freewheeling diode (not a half bridge):

```
P_Coss = E_oss * f_sw
```

Only the MOSFET's stored Coss energy is dissipated at turn-on. There is no complementary FET whose Coss must be charged from the bus. The diode's junction capacitance (Cj) contributes a similar but typically smaller loss.

### Where to Get E_oss and Q_oss

- **Best:** Read E_oss and Q_oss directly from the datasheet (specified at a given V_DS)
- **Good:** Integrate the Coss vs V_DS curve numerically from the datasheet graph
- **Acceptable:** Use Co(er) from the datasheet: E_oss = 0.5 * Co(er) * V_DS^2
- **WRONG:** Use Coss at a single operating point with 0.5*C*V^2

## 5. Reverse Recovery Loss (Qrr)

**Source:** ST AN5028; Erickson & Maksimovic Ch. 4; Fairchild/onsemi AN-4110

### The Reverse Recovery Problem

When a diode (body diode or discrete) carrying forward current is forced to block reverse voltage, it does not turn off instantly. Stored minority carriers must be swept out, causing a brief reverse current spike. This reverse current flows through the opposing switch (the one turning on), causing additional overlap loss.

### Reverse Recovery Loss Formula

**Factor of 0.5 vs 1.0 debate:**

The commonly cited formula is:

```
P_Qrr = Q_rr * V_reverse * f_sw        [Factor = 1.0]
```

Some references use:

```
P_Qrr = 0.5 * Q_rr * V_reverse * f_sw  [Factor = 0.5]
```

**Resolution:** The factor depends on what loss you are calculating:

- **Factor = 1.0:** Total energy dissipated in the circuit due to reverse recovery. This is the correct factor for total system loss budgeting. The turning-on FET sees the full Qrr as additional current during the turn-on transition, and this current flows through the full bus voltage. Per Erickson (Ch. 4): E_rr = Q_rr * V_bus.

- **Factor = 0.5:** The energy dissipated in the diode itself during the reverse recovery event, assuming a triangular reverse current waveform: E_diode = 0.5 * I_RM * V_R * t_b = 0.5 * Q_rr * V_R (approximately). This is relevant only for diode thermal calculations.

- **For system efficiency calculations, use factor = 1.0.** The total energy extracted from the bus during reverse recovery is Q_rr * V_bus. Some is dissipated in the diode, some in the FET channel, and some in parasitic resistances -- but all of it is lost.

### Practical Complications

1. **Qrr is highly temperature-dependent.** Datasheet Qrr at 25C can be 2-3x lower than at 125C junction temperature. Always use hot Qrr.

2. **Qrr depends on di/dt.** Higher di/dt (faster turn-on) reduces Qrr but increases the peak reverse recovery current I_RM, causing worse voltage overshoot. There is an optimum.

3. **Qrr depends on forward current.** Higher IF before turn-off means more stored charge.

4. **Softness factor matters.** A "snappy" diode (low softness factor S = t_b/t_a) concentrates the recovery energy in a short time, causing high dv/dt and EMI. A "soft" diode spreads the loss but has higher total Qrr.

### By Device Technology

| Technology | Typical Qrr | Reverse Recovery Impact |
|------------|-------------|------------------------|
| Si body diode (planar) | 50-500 nC | Moderate, manageable with snubbers |
| Si body diode (superjunction) | 1-30 uC | VERY HIGH -- dominates switching loss above 100 kHz |
| Si Schottky | ~0 (majority carrier) | Negligible, but limited to <200V |
| Si ultrafast rectifier | 20-100 nC | Moderate, use for high-voltage rectification |
| SiC Schottky (SBD) | ~0 | Negligible, works to 1700V |
| SiC MOSFET body diode | 50-200 nC | Low, and nearly temperature-independent |
| GaN HEMT | 0 (no body diode) | Zero -- major GaN advantage |

## 6. Gate Drive Loss

**Source:** EPC AN030; TI SLYT664

### Equation

```
P_gate = Q_G * (V_drv_on - V_drv_off) * f_sw
```

Where:
- Q_G: Total gate charge (from datasheet, at specified V_GS and V_DS)
- V_drv_on: Gate driver output high voltage
- V_drv_off: Gate driver output low voltage (0 for Si MOSFET, -3V to -5V for GaN)
- f_sw: Switching frequency

### Notes

- This power is dissipated in the gate drive circuit (pull-up/pull-down resistors, driver IC internal resistance, MOSFET internal gate resistance)
- It is NOT dissipated in the MOSFET itself (the gate is capacitive)
- At MHz frequencies, gate drive loss becomes a dominant loss term
- For GaN: Q_G is 5-10x lower than Si at similar ratings -- critical advantage at high frequency

---

## 7. Dead Time Loss (Half-Bridge Only)

**Source:** EPC WP012; Lidow et al.

### Equation

```
P_dead = V_SD * I_load * t_dead * f_sw * 2    [*2 for both transitions]
```

Where:
- V_SD: Body diode (or reverse channel) forward voltage drop during dead time
- I_load: Load current flowing through the body diode
- t_dead: Dead time duration per transition
- f_sw: Switching frequency

### Notes

- For Si MOSFETs: V_SD ~ 0.7-1.2 V (body diode)
- For GaN HEMTs: V_SD ~ 2-5 V (reverse channel conduction, no body diode) -- much higher!
- GaN dead time must be minimized (< 20 ns ideal) to avoid excessive reverse conduction loss
- ZVS eliminates most dead time loss because the inductor current charges Coss during dead time

## 8. IGBT Switching Loss Calculation

**Source:** Semikron AN 1403; Infineon IGBT Datasheet Explanation; Renesas IGBT Loss Calculation AN

IGBTs are fundamentally different from MOSFETs for switching loss because they have minority-carrier tail current. IGBT datasheets specify switching energy directly.

### Datasheet Method (Recommended)

IGBT datasheets provide E_on and E_off curves vs. collector current I_C at a reference voltage V_CC_ref and reference temperature T_j_ref.

**Scaling formula (Semikron):**

```
E_sw(I_C, V_CC, T_j) = E_sw_ref * (I_C / I_C_ref)^K_i * (V_CC / V_CC_ref)^K_v * [1 + TC_sw * (T_j - T_j_ref)]
```

Where:
- E_sw_ref: Switching energy from datasheet at reference conditions
- K_i: Current exponent (typically ~1.0 for IGBT, ~0.5-0.6 for freewheeling diode)
- K_v: Voltage exponent (typically ~1.2-1.4 for IGBT, ~0.6 for diode)
- TC_sw: Temperature coefficient (~0.003 /K for IGBT, ~0.005-0.006 /K for diode)

**Total IGBT switching power loss:**

```
P_sw_IGBT = (E_on + E_off) * f_sw
```

**Total freewheeling diode switching loss:**

```
P_sw_diode = E_rr * f_sw
```

Where E_rr is the diode reverse recovery energy (also scaled using the formula above with diode-specific exponents).

### Integration Limits for E_on and E_off

Per IEC 60747-9 and Semikron convention:
- E_on: Integrated from 10% of I_C rise to 10% of V_CE fall (or 2% depending on manufacturer)
- E_off: Integrated from 10% of V_CE rise to 2% of I_C fall
- **Caution:** Different manufacturers use slightly different integration limits. When comparing devices from different vendors, this can cause 5-15% discrepancy.

### Key Differences from MOSFETs

1. **Tail current:** IGBTs have a slow current tail at turn-off due to minority carrier recombination. This makes E_off much larger relative to E_on compared to MOSFETs.
2. **E_on and E_off scale differently with temperature.** E_on may decrease slightly with temperature (due to lower V_CE(sat) during overlap), while E_off increases significantly.
3. **Gate resistance affects E_on/E_off strongly.** Datasheet values are at a specific R_G. Scaling to different R_G requires additional curves or interpolation.
4. **No Coss model needed** -- the switching energies E_on and E_off already include all capacitive effects in the test circuit.

## 9. GaN-Specific Switching Loss Model

**Source:** EPC AN030; Lidow et al., Ch. 4; TI "GaN to the Rescue" series

### Key Differences from Si MOSFETs

1. **Zero reverse recovery (Qrr = 0):** GaN HEMTs have no body diode, so there is no reverse recovery charge. The P_Qrr term is identically zero. This is the single largest GaN advantage in hard-switched half bridges.

2. **No Miller plateau (in many devices):** Due to very low C_GD/C_GS ratio and fast gate charging, many GaN FETs do not exhibit a distinct Miller plateau. The gate voltage continues rising through the voltage transition. This means the standard Q_GS2/Q_GD model does not directly apply.

3. **Coss loss dominates at high frequency:** With crossover loss and Qrr loss greatly reduced, Coss charging loss (P_Coss = V_bus * Q_oss * f_sw) becomes the dominant switching loss in hard-switched GaN designs.

4. **Dynamic R_DS(on):** GaN devices exhibit current collapse -- R_DS(on) increases transiently after hard switching. This adds a few percent to conduction losses that is not captured by DC R_DS(on). It improves with device maturity but must be accounted for.

### GaN Hard-Switching Loss Model

```
P_sw_GaN = P_cross_on + P_Coss + P_gate + P_dead
```

Where:
- P_cross_on: V-I overlap loss during turn-on (very small, 1-5 ns transition)
- P_cross_off: Negligible for GaN (channel turns off before significant VDS rise)
- P_Coss = V_bus * Q_oss(V_bus) * f_sw  [half bridge]
- P_Qrr = 0
- P_gate = Q_G * V_drv_swing * f_sw
- P_dead = V_SD * I_load * t_dead * f_sw * 2

### GaN Soft-Switching (ZVS) Loss Model

```
P_sw_GaN_ZVS = P_gate + P_Coss_hyst
```

Where:
- P_cross = 0 (ZVS eliminates overlap)
- P_Coss = 0 for ideal ZVS (Coss energy is recovered to load)
- P_Coss_hyst: Coss hysteresis loss (see Section 10 -- typically small for GaN, but nonzero)
- P_gate = Q_G * V_drv_swing * f_sw
- P_Qrr = 0

## 10. SiC MOSFET Switching Loss Model

**Source:** Rohm SiC Application Note; Infineon CoolSiC documentation; Wolfspeed app notes

### Key Differences from Si MOSFETs

1. **Low Qrr body diode:** SiC MOSFET body diodes have much lower Qrr than Si (50-200 nC vs 1-30 uC for superjunction). Furthermore, SiC Qrr is nearly temperature-independent (unlike Si where Qrr can 3-5x from 25C to 150C).

2. **Fast switching capability:** SiC can switch in 10-20 ns (similar to GaN), but gate drive voltage is higher (+15V/-5V typical) and gate oxide is more sensitive to overshoot.

3. **Coss is well-behaved:** SiC MOSFET Coss does not exhibit the severe hysteresis seen in Si superjunction devices (see Section 11).

4. **Higher gate charge than GaN:** SiC typically has 2-5x higher Q_G than GaN at similar ratings, making gate drive loss more significant at high frequency.

### SiC Hard-Switching Loss Model

Use the same gate-charge-based model (Model B) as Si MOSFETs, with these adjustments:

```
P_sw_SiC = P_cross_on + P_cross_off + P_Coss + P_Qrr + P_gate
```

Where:
- P_cross: Use Model B with SiC gate charge parameters
- P_Coss = E_oss * f_sw [single switch] or V_bus * Q_oss * f_sw [half bridge]
- P_Qrr: Use factor 1.0, but Qrr is much smaller than Si superjunction
- P_gate = Q_G * (V_drv_on - V_drv_off) * f_sw, noting V_drv_on = +15..+18V, V_drv_off = -2..-5V

### SiC vs Si Superjunction Switching Loss Comparison

At 400V bus, 20A, 100 kHz, typical values:

| Loss Component | Si Superjunction | SiC MOSFET |
|---------------|-----------------|------------|
| P_cross_on    | 2-5 W           | 1-3 W      |
| P_cross_off   | 1-3 W           | 0.5-2 W    |
| P_Coss        | 1-3 W           | 0.5-2 W    |
| P_Qrr (body diode in half bridge) | 5-50 W (!) | 0.5-2 W |
| P_gate        | 0.1-0.3 W       | 0.2-0.5 W  |
| **Total switching** | **10-60 W** | **3-9 W** |

The SiC advantage is overwhelmingly from the Qrr reduction. This is why SiC dominates in hard-switched high-voltage applications (PFC, inverters, DC-DC).

## 11. Coss Hysteresis Loss in ZVS Applications

**Source:** Fedison & Harrison, APEC 2016; Vishay AN-2017 "Beware of Zero Voltage Switching"; MDPI Energies 2020

### The Problem

In soft-switched (ZVS) converters, textbook analysis says Coss loss is eliminated because the energy stored in Coss is recovered to the load. This is NOT fully correct for Si superjunction MOSFETs.

Advanced superjunction MOSFETs exhibit hysteresis in their output capacitance -- the Coss vs VDS curve is different for charging vs discharging. This means energy is dissipated inside the device during each Coss charge/discharge cycle even under perfect ZVS.

### Magnitude

- For standard planar MOSFETs: Coss hysteresis loss is negligible
- For superjunction MOSFETs: Coss hysteresis can dissipate 5-30 uJ per cycle, which at 100-500 kHz becomes 0.5-15 W -- often MORE than the conduction loss
- For SiC MOSFETs: Coss hysteresis is small (well-behaved capacitance)
- For GaN HEMTs: Coss hysteresis is very small

### Measurement

Coss hysteresis cannot be measured with a standard impedance analyzer. It requires a large-signal Sawyer-Tower measurement circuit. Traditional hard-switching double-pulse tests also do not capture it correctly.

### Design Impact

For LLC and other ZVS resonant converters:
- Si superjunction MOSFETs may have unexpectedly high losses at light load due to Coss hysteresis
- SiC and GaN are fundamentally better choices for ZVS converters above ~200 kHz
- Coss hysteresis loss is proportional to frequency and INDEPENDENT of load current
- It sets a minimum loss floor that cannot be reduced by circuit design

## 12. Common Source Inductance Effects

**Source:** TI SLPA009A; Infineon design notes

### What It Is

Common source inductance (L_CS) is the parasitic inductance shared between the power loop and the gate drive loop. It typically comes from the source bond wires and package lead.

### Effect on Switching Loss

L_CS provides negative feedback during switching:

```
V_GS_effective = V_G - V_S = V_drv - I_G*R_G - L_CS * di_D/dt
```

During turn-on, rising di_D/dt creates a voltage drop across L_CS that opposes the gate drive, slowing down turn-on. During turn-off, falling di_D/dt speeds up turn-off.

**Net effect:** L_CS increases turn-on loss and decreases turn-off loss. Since turn-on loss is typically larger (due to Qrr), the net effect is increased total switching loss.

### Typical Values

- TO-220: 3-7 nH
- D2PAK: 2-5 nH
- SO-8: 1-2 nH
- TI NexFET PowerBlock: 0.1-0.3 nH (Kelvin source connection)
- GaN BGA: <0.1 nH

### Impact on Accuracy

At high di/dt (>1 A/ns), even 1 nH of L_CS can add 10-30% to switching loss compared to the gate-charge model prediction. This is why the gate-charge-based model (Model B) systematically underestimates losses for leaded packages.

## 13. Model Selection Guide

| Situation | Recommended Model | Expected Accuracy |
|-----------|-------------------|-------------------|
| Quick topology comparison | Model A (piecewise linear) | +/-50% |
| Design-level Si MOSFET loss | Model B (gate charge) + Coss + Qrr | +/-20-30% |
| Design-level GaN loss | EPC AN030 method (Qoss-based) | +/-15-25% |
| Design-level SiC loss | Model B + Coss + Qrr (low) | +/-20-30% |
| Design-level IGBT loss | Datasheet Eon/Eoff with scaling | +/-10-20% |
| ZVS converter (Si SJ) | Coss hysteresis measurement | Must measure |
| ZVS converter (SiC/GaN) | Gate loss + small Coss hysteresis | +/-10-15% |
| Final validation | Double-pulse test or full circuit measurement | +/-5% |

### Hierarchy of Accuracy

From least to most accurate:
1. Piecewise-linear overlap (Model A) -- worst, but simple
2. Gate-charge-based (Model B) -- good for component selection
3. Model B + Coss integral + Qrr (hot) -- good for efficiency prediction
4. Model B + Coss + Qrr + L_CS correction -- best analytical model
5. SPICE simulation with nonlinear device models -- best pre-hardware
6. Double-pulse test measurement -- ground truth for switching energy
7. Full converter calorimetric measurement -- absolute ground truth

## 14. Practical Calculation Checklist

When calculating switching losses for a design:

1. **Identify switching conditions:** V_DS at switching, I_D at switching, f_sw. Remember these may differ at V_in_min vs V_in_max.

2. **Calculate crossover loss** using Model B (gate charge method). Use the actual gate driver voltage and total gate resistance.

3. **Add Coss loss:**
   - Half bridge: P_Coss = V_bus * Q_oss(V_bus) * f_sw
   - Single switch: P_Coss = E_oss(V_DS) * f_sw
   - Use datasheet E_oss/Q_oss or integrate Coss curve. NEVER use 0.5*Coss*V^2 with a single Coss value.

4. **Add Qrr loss:**
   - P_Qrr = Q_rr(hot) * V_bus * f_sw
   - Use Qrr at maximum expected junction temperature (125C or 150C)
   - For SiC body diode: Qrr is small and temperature-stable
   - For GaN: Qrr = 0
   - For Si superjunction body diode: Qrr can be enormous -- consider external SiC Schottky

5. **Add gate drive loss:** P_gate = Q_G * V_drv_swing * f_sw

6. **Add dead time loss (half bridge):** P_dead = 2 * V_SD * I_load * t_dead * f_sw

7. **Check at worst case:** Switching loss is typically worst at maximum V_in (highest V_DS) and maximum I_out (highest I_D). Check both corners.

8. **Reality check:** If calculated switching loss is <10% of total loss, the design is switching-loss-limited potential is low. If >40%, consider soft switching or higher-performance devices.

## 15. Summary of Key Equations

### Hard-Switched Si MOSFET (Complete)

```
P_total_switching = P_cross + P_Coss + P_Qrr + P_gate + P_dead

P_cross   = 0.5 * V_bus * I_load * (Q_GS2 + Q_GD) * (1/I_G_on + 1/I_G_off) * f_sw
P_Coss    = E_oss(V_bus) * f_sw                          [single switch]
          = V_bus * Q_oss(V_bus) * f_sw                   [half bridge]
P_Qrr     = Q_rr(T_j_max) * V_bus * f_sw
P_gate    = Q_G * (V_drv_on - V_drv_off) * f_sw
P_dead    = 2 * V_SD * I_load * t_dead * f_sw            [half bridge only]
```

### Hard-Switched GaN HEMT

```
P_total_switching = P_cross_on + P_Coss + P_gate + P_dead

P_cross_on ~ very small (1-5 ns transitions)
P_Coss     = V_bus * Q_oss(V_bus) * f_sw                 [half bridge, dominant term]
P_Qrr      = 0
P_gate     = Q_G * (V_drv_on - V_drv_off) * f_sw
P_dead     = 2 * V_SD_GaN * I_load * t_dead * f_sw       [V_SD_GaN is high: 2-5V!]
```

### Hard-Switched SiC MOSFET

```
P_total_switching = P_cross + P_Coss + P_Qrr + P_gate

P_cross   = 0.5 * V_bus * I_load * (Q_GS2 + Q_GD) * (1/I_G_on + 1/I_G_off) * f_sw
P_Coss    = E_oss(V_bus) * f_sw  or  V_bus * Q_oss(V_bus) * f_sw
P_Qrr     = Q_rr * V_bus * f_sw                          [small and temp-stable]
P_gate    = Q_G * (V_drv_on - V_drv_off) * f_sw          [V_drv swing is 18-23V!]
```

### IGBT (Using Datasheet Energies)

```
P_sw_IGBT  = (E_on + E_off) * f_sw
P_sw_diode = E_rr * f_sw

E_sw = E_sw_ref * (I_C/I_C_ref)^K_i * (V_CC/V_CC_ref)^K_v * [1 + TC_sw*(T_j - T_j_ref)]
```

### ZVS Condition (Half Bridge)

```
ZVS achieved when:  0.5 * L_mag * I_ZVS^2 >= Q_oss(V_bus) * V_bus
                    equivalently: 0.5 * L * I^2 >= E_oss(V_bus) + 0.5 * C_oss_partner * V_bus ... [use Q_oss method]

Simplified:         L * I_ZVS^2 >= 2 * Q_oss * V_bus
```

The inductor energy must be sufficient to fully discharge the turning-on FET's Coss AND charge the turning-off FET's Coss, against the bus voltage.
